# Chemotaxis Simulation Visualizer Suite

A modular PyQt5 / PyVista / Matplotlib application for visualizing
bacterial chemotaxis simulations in cylindrical geometry, paired with
a C++ simulation engine and batch experiment runner.

## Running Simulations

The simulation code (C++ models) lives alongside the visualizer in the root folder. It is independent of the Python visualizer - you can compile and run it separately.

### Requirements

**C++ compiler:**
- Windows: [MinGW-w64](https://www.mingw-w64.org/) with `g++` (the same compiler used by the build script). Easiest to install via [MSYS2](https://www.msys2.org/) or [WinLibs](https://winlibs.com/).
- Linux: `g++` from your package manager (`sudo apt install build-essential`)
- macOS: `g++` via Xcode Command Line Tools (`xcode-select --install`)

**Python (for the experiment runner):**
The `run_experiments.py` script only needs the `toml` package - it auto-installs it if missing. You can also install it manually:
```bash
pip install toml
```
This can be done inside the same venv as the visualizer, or separately.

### Step 1 - Configure parameters

Edit `constants.hpp` to set grid size, geometry and time parameters:

```cpp
const double R = 5.0;      // cylinder radius
const double H = 10.0;     // cylinder height
const int N_RHO = 40;      // radial grid points
const int N_PHI = 224;     // angular grid points
const int NZ    = 80;      // vertical grid points
const double T_FINAL = 400.0;
```

Physical parameters (D_U, CHI, ALPHA etc.) can be set in `constants.hpp` as defaults, or overridden per-experiment in `experiments.toml`.

---

### Step 2 - Build

**Windows:**
```bat
build.bat all
```

**Linux / macOS:**
```bash
g++ -O3 -march=native -o bin/1d      1D.cpp      writer.cpp
g++ -O3 -march=native -o bin/top     2Dtop.cpp   writer.cpp
g++ -O3 -march=native -fopenmp -o bin/side   2Dside.cpp  writer.cpp
g++ -O3 -march=native -fopenmp -o bin/3d     3D.cpp      writer.cpp
g++ -O3 -march=native -o bin/coupled 2Dcombined.cpp writer.cpp
g++ -O3 -march=native -o bin/ic_gen  initialConditions.cpp
mkdir -p bin results ic
```

---

### Step 3 - Generate initial conditions

```bat
bin\ic_gen.exe all          # Windows
bin/ic_gen all               # Linux / macOS
```

This generates `ic/master_ic.dat` (3D random IC) and extracts consistent 2D and 1D ICs from it. Repository already contains default initial conditions which were used for uploaded experiments (see experiment_results)

---

### Step 4 - Define experiments

Edit `experiments.toml` to add or modify parameter sets:

```toml
[[experiment]]
name = "my_experiment"
description = "Testing higher chemotaxis"
D_U   = 0.1
CHI   = 9.0
ALPHA = 1.0
BETA  = 0.73
GAMMA = 0.025
W_0   = 1.0
```

---

### Step 5 - Run experiments

```bash
# Run all experiments, all models
python run_experiments.py

# Dry run - see what would be executed without running
python run_experiments.py --dry-run

# Run only specific experiments (by number)
python run_experiments.py --experiments 1 3

# Run only specific models
python run_experiments.py --models 2D_polar 3D

# Skip recompilation (reuse existing binaries)
python run_experiments.py --skip-compile

# Custom config and output directory
python run_experiments.py --config my_params.toml --output-dir my_results
```

Results are saved to `experiment_results/runXX/<timestamp>_<name>/`:
- `results/` - snapshot `.dat` files (load into the visualizer)
- `spatiotemporal/` - spatiotemporal data `.dat` files
- `metadata/` - logs and JSON summaries per experiment

---

### Running a single model manually

Without the batch runner, you can run models directly:

```bat
bin\1d.exe      experiments.toml baseline ic\ic_1d.dat
bin\top.exe     experiments.toml baseline ic\ic_polar.dat
bin\side.exe    experiments.toml baseline ic\ic_cyl.dat
bin\3d.exe      experiments.toml baseline ic\master_ic.dat
bin\coupled_2d.exe experiments.toml baseline ic\ic_polar.dat ic\ic_cyl.dat
```

Output `.dat` files appear in the `results/` folder and can be opened directly in the visualizer.

python3 -m venv venv

- If `pip install` fails on Windows, upgrade pip first:
  ```bat
  python -m pip install --upgrade pip
  ```

  ```bat
  venv\Scripts\activate   # Windows
  source venv/bin/activate  # Linux / macOS
  ```

## Supported Data Formats

All data files are plain-text `.dat` files with a **header section**
followed by **snapshot blocks**.

### Header format

Lines beginning with `#` are parsed as key–value pairs:

```
# KEY = VALUE
```

Blank lines and comment-only lines are ignored.  
The file type (polar / cylindrical / 3D volume) is detected automatically
from the header keys present.

---

### 2D Polar file (top-disk view)

Represents a flat disk in polar coordinates (r, φ).

**Required header keys:**

| Key | Description |
|-----|-------------|
| `N_RHO` | Number of radial grid points |
| `N_PHI` | Number of angular grid points |
| `R` | Outer radius |

**Data blocks:**

```
SNAPSHOT=<time>
<N_RHO rows, each with N_PHI space-separated float values>
```

Each row is one radial level; columns are angular positions 0 → 2π.

---

### 2D Cylindrical file (side-surface view)

Represents the unwrapped cylinder surface in (φ, z) coordinates.

**Required header keys:**

| Key | Description |
|-----|-------------|
| `N_PHI` | Number of angular grid points |
| `NZ` | Number of vertical grid points |
| `R` | Cylinder radius |
| `H` | Cylinder height |

**Data blocks:**

```
SNAPSHOT=<time>
<NZ rows, each with N_PHI space-separated float values>
```

Each row is one height level (z); columns are angular positions 0 → 2π.

---

### 3D Volume file

Full cylindrical volume in (ρ, φ, z) coordinates.

**Required header keys:**

| Key | Description |
|-----|-------------|
| `N_RHO` | Number of radial grid points |
| `N_PHI` | Number of angular grid points |
| `NZ` | Number of vertical grid points |
| `R` | Cylinder radius |
| `H` | Cylinder height |

**Data blocks:**

```
SNAPSHOT=<time>
<N_RHO × NZ rows, each with N_PHI space-separated float values>
```

Rows cycle through (rho_0, z_0), (rho_0, z_1), …, (rho_1, z_0), …

---

### Spatiotemporal file

A 2D spatiotemporal diagram saved as a plain matrix (no `SNAPSHOT` blocks).

**Recommended header keys** (optional but used for correct axis scaling):

| Key | Description |
|-----|-------------|
| `T` | Total simulation time |
| `L` | Spatial domain length |

**Data:** one row per time sample, one column per spatial point.  
Expected filename convention: `<base>_spatiotemporal.dat` alongside
the corresponding snapshot file - the polar and cylindrical viewers
load it automatically when a snapshot file is opened.

---

### Legacy naming support

Older experiment files (before major refactoring) using `NX` instead of `N_PHI`, or `ERDVE_RHO` / `ERDVE_PHI`
instead of `N_RHO` / `N_PHI`, are detected and loaded automatically.

---

### Model parameters (optional, shown in UI)

The following header keys are recognised and displayed in the info panels
and hover tooltips:

| Key | Description |
|-----|-------------|
| `D_U` | Cell diffusion coefficient |
| `CHI` | Chemotactic sensitivity |
| `ALPHA` | Cell birth rate |
| `BETA` | Cell death rate |
| `D_W` | Chemical diffusion coefficient |
| `GAMMA` | Chemical decay rate |
| `W_0` | Initial chemical concentration |
| `dt` | Time step |
| `N_steps` | Total number of simulation steps |
| `SAVE_EVERY_T` | Time interval between saved snapshots |

All other `# KEY = VALUE` header lines are loaded and stored but not
specifically labelled in the UI.

## Application Modules

| Tab | Description |
|-----|-------------|
| **2D Polar** | Snapshot viewer for top-disk polar data. Colormap, GIF export. |
| **2D Side** | Snapshot viewer for cylindrical side-surface data. Colormap, GIF export. |
| **3D Combined** | Interactive PyVista view of disk + cylinder surface together. GIF export. |
| **3D Volume** | Full volumetric rendering with orthogonal slices, isosurface layers, GIF export. |
| **Spatiotemporal** | Multi-panel spatiotemporal diagram grid. Hover shows model parameters. |
| **2D Experiment Comparer** | Side-by-side grid of polar and/or cylindrical snapshots from multiple experiments, with parameter diff table. |
